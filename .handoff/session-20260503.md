## 3. 마스터 누적 결정 30+개 (시간순, 이유 포함)

- 본질을 메시징에서 기억 인프라로 재정의 — 마스터가 원하는 것은 "어디서든 같은 AI"이지 메시지 전달이 아님
- 5층 메모리 채택 (L0~L4) — 원시 메시지부터 정체성까지 계층 분리해야 회상 품질과 프라이버시 제어 가능
- 사이드카 = 머신마다 1개, Rust 단일 바이너리 — 설치 단순화, 의존성 없음
- 모드 4종: daemon / TUI / GUI / headless — 서버·터미널·데스크탑·CI 모두 대응
- 키페어 옵션 B 채택 (서브에이전트 자동 발급, HD 파생) — 마스터 시드 1개에서 모든 키를 파생, 관리 단순화
- Transport 자동 라우팅 (localhost > Tailscale > XMTP) — 네트워크 상황 따라 최적 경로 자동 선택, fallback 아님 (명시적 단계)
- Discord 기본 ON, 봇 1개 + Webhook 발신자 분리 (모델 C) — 마스터가 이미 12+ 채널 운영 중, Webhook으로 발신자 분리 가능한 거의 유일한 옵션
- Telegram 기본 ON, Setup Agent 1:1 — 마스터가 이미 사용 중인 채널, 온보딩 단계에서 Setup Agent가 1:1 안내
- 데이터 디렉토리 ~/.openxgram/ — 표준 Unix dotfile 위치, 다른 경로 금지 (혼동 방지)
- 이메일 어댑터 영구 제외 — 마스터 요청, 복잡도 대비 효용 없음
- 코드 추출 LLM 어댑터 패턴 채택 — Gemini/Claude/OpenAI/Ollama/Template 사용자 선택, 특정 모델 종속 금지
- Inbound webhook 포트 14921 — Starian 서비스 포트 체계와 충돌 없음
- Rate limit 분당 1000건 — 안전망, 실제 사용보다 훨씬 높아서 운영 방해 없음
- on-schedule 온보딩에서 결정 — 스케줄 설정을 온보딩 단계에서 대화형으로
- OS 등록 [Y/n] 대화형 — 강제 등록 금지, 사용자 동의 필수
- BIP39 24단어 + Y/N 확인 + 재입력 옵션 권장 — 시드 분실 방지, 단순 복사붙여넣기 오류 차단
- Linux trash 크레이트 채택 — rm -rf 대신 복구 가능한 삭제
- 흔적 검사 홈만 기본 — 전체 파일시스템 스캔은 너무 느리고 위험
- backup 위치 ~/.openxgram/backup/ — 데이터 디렉토리 내 일관성
- 다른 머신 sync = Tailscale mTLS + 시드 서명 이중 — 전송 암호화 + 메시지 무결성 검증
- 라이선스 MIT — 오픈소스 호환성 최대화, MVP 후 public 전환 용이
- GitHub 조직 OpenXgram (private, MVP 후 public) — 조직 단위로 관리, 마스터 계정 분리
- 표준 문서 4종 (LICENSE/CONTRIBUTING/CoC/SECURITY) — GitHub best practice, 오픈소스 전환 준비
- CI = GitHub Actions — 별도 인프라 불필요, GitHub 조직에서 즉시 사용
- DB rusqlite — 순수 Rust, 단일 파일 SQLite, 크로스 플랫폼
- 임베딩 multilingual-e5-small (한국어) — BGE-small 대비 한국어 성능 우수, 마스터 사용 언어 최적화
- 암호화 ChaCha20-Poly1305 — AES-GCM보다 소프트웨어 구현 빠름, ARM/모바일 최적
- XMTP는 reqwest로 REST 직접 (공식 Rust SDK 부재) — Rust SDK 없음, REST API 직접 호출이 유일한 옵션
- SSH 키 분리 (id_ed25519_openxgram) — 공용 키와 절대 분리, 보안 격리
- Memory Transfer 백업 채널 3종 (클립보드/Telegram/Discord) — 단일 채널 장애 시 대안 확보
- Lifecycle Manager 6명령 (init/status/doctor/reset/migrate/uninstall) — 운영에 필요한 최소 명령 집합
- 도메인 openxgram.org (Cloudflare) — 도메인 선점, Phase 2 공개 시 사용
- Akashic은 임시 미사용, Starian 메모리에 자동 기록 — OpenXgram MVP 완료 후 Vault로 마이그레이션 예정
