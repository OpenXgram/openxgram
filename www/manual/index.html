OpenXgram 사용 설명서 (일반 사용자용)

OpenXgram user manual (for everyone)

이 페이지는 일반 사용자용입니다 — 코드 없이 GUI 만으로 OpenXgram 을 쓰는 방법을 안내합니다. 화면 그림을 보면서 어떤 버튼이 무엇을 하는지, 비유와 짧은 문장으로 풀어 썼습니다. 처음이라면 각 탭의 💡 처음이라면 박스만 따라 해도 5 분 안에 첫 액션이 가능합니다.

This page is for everyday users — use OpenXgram via the GUI, no code required. For each screen we explain what every button does using analogies and short sentences. New here? Just follow the 💡 First time? box in each tab and you'll be doing your first action within 5 minutes.

※ 그림은 모두 실제 데스크톱 앱(Tauri) 컴포넌트를 그대로 화면에 그린 결과입니다. 보이는 글자·숫자는 샘플 데이터 라서 실제 사용 시엔 본인 데이터가 보입니다. 데스크톱 앱과 시각적으로 똑같이 보이도록 만들었습니다 (같은 렌더링 엔진).

Note — every image shows the real desktop app (Tauri) components rendered as-is. The text and numbers are sample data — your own data will appear in real use. Visually identical to the desktop app (same rendering engine).

3 가지 사용 방식 · 같은 데이터

사용 방식 고르기 — CLI · TUI · GUI

Pick your interface — CLI · TUI · GUI

OpenXgram 은 한 곳에 데이터를 두고, 들어가는 문(門)이 3 개입니다. 어느 문으로 들어가도 본 데이터·결과는 같습니다. 은행 — 창구 / ATM / 모바일 앱 처럼, 상황에 맞는 걸 고르세요.

OpenXgram keeps data in one place, with three doors into it. Whichever door you take, the data and result are the same. Bank — counter / ATM / mobile app — pick what fits the moment.

CLI

xgram <cmd> — 자동화·반복 작업용

xgram <cmd> — for automation and scripts

  • 스크립트와 AI 도구(Claude Code · Cursor · Aider)에 바로 연결
  • Plug into scripts and AI tools (Claude Code · Cursor · Aider)
  • SSH·서버에서도 그대로 동작
  • Works over SSH and on servers
  • 다른 명령과 입출력을 이어 붙일 수 있음
  • Compose with other commands

TUI

xgram tui — 터미널 안 작은 대시보드

xgram tui — a small dashboard inside the terminal

  • 화면(GUI) 환경이 없는 서버에서 유용
  • Useful on servers without a graphical screen
  • 설치·연결 상태 빠른 확인
  • Quick install / connection check
  • 키 입력 Q / Esc 로 끝내기
  • Exit with Q / Esc

GUI

데스크톱 앱 — 9 탭으로 클릭 사용

Desktop app — click through 9 tabs

  • 처음 사용자 · 봇 연결 · 라이브 채널 보기에 가장 편함
  • Best for first-time users, bot setup, live channel views
  • 비밀값은 30 초 동안만 잠깐 보여주는 안전 모드
  • Secrets unlock for 30 seconds only — safe-mode reveal
  • 같은 데이터 · 같은 결과
  • Same data, same result
실용 팁 — Practical tip — 처음 한 번은 GUI 로 봇·신원을 끝내고, 평소엔 CLI 로 자동화. 서버 점검은 TUI. Set up bots and identity once via GUI, automate daily flow with CLI, peek at servers via TUI.

탭 1 · 시작하기 / Onboarding

1. 시작하기 — 첫 사용 안내

1. Onboarding — first-time guide

이 탭의 역할 — 처음 들어왔을 때 인사하고, 준비가 됐는지 확인한 뒤, 다음 단계(기억 탭)로 데려갑니다. 길잡이 / onboarding

What it does — greets you on first launch, checks that everything is ready, then guides you to the next step (Memory tab). A guide

처음이라면: First time? 이 한 가지만 — 터미널에서 xgram init --alias 내이름 입력하고, GUI 의 "다시 확인" 버튼 한 번. 그러면 끝입니다. Just one thing — run xgram init --alias yourname in your terminal, then click "Re-check" in the GUI once. Done.

어떻게 쓰나요

How to use

  • 터미널에서 xgram init --alias 내이름 을 실행합니다. (내 신원과 데이터 파일이 만들어집니다.)
  • Run xgram init --alias yourname in a terminal. (Creates your identity and data file.)
  • 화면에 12 개의 영어 단어가 보입니다 — 종이에 적어 안전한 곳에 보관하세요. 통장 비밀번호 같은 복구용 단어
  • Twelve English words appear — write them on paper and keep them safe. Like a bank-recovery passphrase
  • "초기화 상태 다시 확인" 버튼을 눌러 초기화 완료 라는 표시가 나오면 성공.
  • Click "Re-check initialization" until you see Initialized.
  • (선택) 알림 탭으로 가서 Discord 나 Telegram 봇을 연결합니다.
  • (Optional) Go to the Notify tab and connect a Discord or Telegram bot.
팁Tip — 이미 시작 설정이 끝나 있으면, 다음에 GUI 를 켜면 자동으로 기억 탭부터 보입니다.If setup is already done, the GUI opens straight to the Memory tab next time.
주의Caution — 12 개 단어는 화면에 그대로 보입니다. 화면 녹화·공유·캡처를 조심하세요.The 12-word recovery shows as plaintext. Beware of screen recording / sharing / screenshots.

같은 작업의 명령어: xgram init --alias 내이름 · xgram status

Same as command: xgram init --alias yourname · xgram status

탭 2 · 기억 / Memory

2. 기억 — AI 의 노트북 검색

2. Memory — search the AI's notebook

이 탭의 역할 — AI 가 기록해 둔 모든 것을 한 입력창으로 검색합니다. AI 의 노트북 안을 단어 한 번으로 찾아내는 곳입니다.

What it does — search everything the AI has noted down, from one box. The AI's notebook — find anything with a single word.

처음이라면: First time? 이 한 가지만 — 검색창에 단어 하나(예: "회의") 입력. 잠깐 멈추면 자동으로 결과가 뜹니다. 체크박스는 일단 그대로 두세요. Just one thing — type a word (e.g. "meeting"). Pause typing and results appear automatically. Leave the checkboxes alone for now.

기억의 5 단계 — 간단한 것부터 복잡한 것 순서입니다.

Five layers of memory — simplest to richest.

  • L0 메시지 — 주고받은 원문 그대로
  • L0 messages — raw conversations
  • L1 에피소드 — 한 세션 묶음
  • L1 episodes — grouped sessions
  • L2 기억 — 사실 · 결정 · 규칙 (별표 가능)
  • L2 memories — facts, decisions, rules (pinnable)
  • L3 패턴 — 새로움 / 반복 / 고정 자동 분류
  • L3 patterns — new / recurring / routine
  • L4 정체성 — AI 의 성격·성향 (밤마다 자동 정리)
  • L4 traits — personality, tendencies (auto-distilled nightly)

어떻게 쓰나요

How to use

  • 위쪽 입력창에 검색어를 입력합니다. 타이핑을 멈추면 자동으로 검색이 시작됩니다.
  • Type a search term in the top box. Search runs automatically when you pause typing.
  • L0 / L2 / L4 체크박스로 어느 단계에서 찾을지 선택합니다 (필요할 때만).
  • Use L0 / L2 / L4 checkboxes to scope which layers to search (only when needed).
  • 결과는 점수가 높은 것부터 위에 나옵니다. 각 줄에 단계 · 점수 · 본문이 보입니다.
  • Top hits show first. Each row shows layer, score, and content.
팁Tip — 대화 흐름을 시간 순으로 보고 싶으면 L0 + L2 만 켜고 검색하세요.To follow a chronological thread, enable only L0 + L2.

같은 작업의 명령어: xgram session recall <검색어>

Same as command: xgram session recall <keyword>

※ 검색은 AI 가 알아듣는 작은 모델(약 120MB)을 한 번 내려받아 사용합니다 — 사용자는 신경 쓸 필요 없습니다.

※ Search uses a small AI model (~120MB) downloaded once — you don't need to manage it.

탭 3 · Vault

3. Vault — 비밀번호 금고

3. Vault — your password safe

이 탭의 역할 — 다른 AI 가 비밀값(API 키 같은 것)을 요청하면 내가 직접 허락/거절합니다. 내 비밀값을 잠깐 꺼내볼 수도 있습니다. 비밀번호 금고 — 내가 직접 관리

What it does — when other agents ask for secrets (like API keys), you approve or deny. You can also peek at your own secrets briefly. A password safe you control

처음이라면: First time? 이 한 가지만 — 위쪽 "대기 중" 카드에 요청이 보이면, 누가·무엇을·언제 요청했는지 확인하고 승인 또는 거부 버튼을 누르세요. Just one thing — when a request appears in the "Pending" card, check who / what / when, then click Approve or Deny.
  • 위쪽 대기 중(Pending) 카드 — 요청자(어떤 AI), 요청한 키 이름, 요청 시각이 보입니다. 확인 후 승인/거부.
  • Top Pending card — see who, which key, when. Then approve or deny.
  • 거부할 땐 사유를 적습니다 (변조 불가능한 기록audit chain에 남습니다).
  • When denying, enter a reason — saved to an immutable logaudit chain.
  • 아래쪽 잠깐 보기(Reveal) 에 키를 입력하면 30 초 동안만 가림이 풀립니다. 시간이 지나면 자동으로 다시 가려집니다.
  • Bottom Reveal — type a key and it unmasks for 30 seconds, then re-masks automatically.
  • 복사 버튼을 누르면 클립보드에 들어가지만, 30 초 후 자동으로 지워집니다.
  • Copy puts the value on the clipboard, then clears it after 30 seconds.
주의Caution — 승인 창에 표시된 지문(고유 신원fingerprint) 이 내가 아는 AI 의 것과 일치하는지 꼭 확인하세요.Always verify the fingerprintunique ID in the dialog matches the agent you trust.

같은 작업의 명령어: xgram vault pending-list · xgram vault approve <id> · xgram vault reveal <key>

Same as command: xgram vault pending-list · xgram vault approve <id> · xgram vault reveal <key>

탭 4 · 피어 / Peers

4. 피어 — 친구 목록 등록

4. Peers — your friend list

이 탭의 역할 — 다른 머신이나 AI 를 친구 목록 에 추가하고, 정말 그 친구가 맞는지 지문으로 확인합니다.

What it does — add other machines or agents to your friend list, then verify by fingerprint that they really are who they say.

처음이라면: First time? 이 한 가지만 — 친구의 이름·주소·공개키를 입력하고 "피어 추가". 다음 화면에 나오는 지문을 친구한테 전화로 불러줘서 같은지 확인하세요. Just one thing — enter alias / address / public key, click "Add peer". On the next screen, read the fingerprint to your friend over the phone and confirm it matches.
  • 이름(별명) · 주소 (http://… 또는 nostr://…) · 공개키(긴 글자열) 를 입력합니다.
  • Enter alias · address (http://… or nostr://…) · public key (a long string).
  • "피어 추가" 누르면 지문(짧은 식별자) 이 뜹니다. 친구한테 전화나 다른 채널로 같은지 확인 → 확정.
  • Click "Add peer" → a fingerprint appears. Verify with the friend on a different channel (phone, paper) → confirm.
  • (선택) "머신" 항목 — 어느 컴퓨터인지 표시해 두면 권한 화이트리스트에 쓸 수 있습니다.
  • (Optional) "machine" — labels which computer; used for permission whitelisting.
주의Caution — 지문 확인 없이 추가하면 가짜 친구가 끼어들 수 있습니다. 의심스러우면 거절하세요.Adding without checking the fingerprint risks an imposter sneaking in. Deny anything you can't verify.

같은 작업의 명령어: xgram peer add --alias eno --address http://… --pubkey 0x… · xgram peer list

Same as command: xgram peer add --alias eno --address http://… --pubkey 0x… · xgram peer list

탭 5 · 알림 / Notify

5. 알림 — Discord / Telegram 봇 연결 마법사

5. Notify — Discord / Telegram bot wizard

이 탭의 역할 — 봇 토큰을 붙여넣으면 자동으로 검사 → 내 채팅 ID 찾기 → 저장 → 테스트 메시지까지, 클릭 한두 번으로 끝납니다.

What it does — paste a bot token and it auto-validates, detects your chat ID, saves, and sends a test — all in one or two clicks.

처음이라면: First time? 이 한 가지만 — Telegram 봇을 만들어 받은 토큰을 붙여넣고 "토큰 검증". 봇한테 /start 한 번 보낸 뒤 "내 chat_id 자동 감지" → "저장 + 테스트". 끝. Just one thing — paste your Telegram bot token, click "Validate", send /start to the bot once, click "Auto-detect chat_id", then "Save + send test". Done.
  • Telegram: BotFather 에서 받은 토큰을 붙여넣고 "토큰 검증" 클릭 → 봇 이름이 표시되면 OK.
  • Telegram: paste BotFather token, click "Validate" → bot username appears.
  • 봇에게 직접 /start 한 번 보낸 뒤 "내 chat_id 자동 감지" 클릭 (봇이 어디로 보낼지 알아냅니다).
  • Send /start to the bot once, then "Auto-detect chat_id" (figures out where to deliver).
  • "저장 + 테스트 전송" 누르면 봇에 "OpenXgram GUI 연결 성공 ✓" 가 도착합니다.
  • Click "Save + send test" → "OpenXgram GUI connected ✓" arrives in the bot.
  • Discord 도 같은 방식 — 채널 ID 또는 Webhook 주소를 추가로 넣을 수 있습니다.
  • Discord follows the same flow — optional channel ID or webhook URL.
팁Tip — 토큰은 자동으로 금고(Vault)에 들어갑니다. 평문 파일이나 환경변수에 두지 마세요.Tokens go into the Vault automatically. Don't keep them in plaintext files / env vars.

같은 작업의 명령어: xgram notify setup-telegram · xgram notify setup-discord

Same as command: xgram notify setup-telegram · xgram notify setup-discord

탭 6 · 채널 / Channel

6. 채널 — 단체 채팅방 라이브

6. Channel — group chat live view

이 탭의 역할 — 단체 채팅방 (AI 들끼리 + 나) 의 상태를 한눈에. 어떤 알림 채널이 연결됐는지, 친구는 몇 명인지, 예약된 메시지 몇 개가 대기 중인지, 최근 20 개 메시지(한국 시간 기준).

What it does — at-a-glance dashboard of your group chat (AIs + you): which notify channels are configured, peer count, pending schedules, latest 20 messages (KST).

처음이라면: First time? 이 한 가지만 — "새로고침" 한 번 클릭. 알림 어댑터 상태와 최근 메시지가 같이 갱신됩니다. Just one thing — click "Refresh" once. Adapter status and recent messages update together.
  • "새로고침" 버튼 — 상태와 최근 메시지를 동시에 갱신합니다.
  • "Refresh" — refetches both status and recent messages.
  • 알림 채널마다 연결됨 / 미설정 표시 확인.
  • Per-channel configured / not set badges.
  • "피어 수" / "예약 대기" 카드 — 단체 채팅방이 얼마나 활발한지 한눈에.
  • "Peers" / "Schedule pending" cards — how busy the hub is, at a glance.

같은 작업의 명령어: xgram channel status · xgram channel recent --limit 20

Same as command: xgram channel status · xgram channel recent --limit 20

탭 7 · 예약 / Schedule

7. 예약 — 예약 발송

7. Schedule — send-later

이 탭의 역할 — 메시지를 정해진 시각에 보내거나, 매일 같은 시간에 반복 발송합니다. 이메일 예약 발송 / 알람 처럼.

What it does — send a message at a chosen time, or recurring on a schedule. Like scheduled email / a recurring alarm.

처음이라면: First time? 이 한 가지만 — 대상 = "역할명", 종류 = "한 번", 시각 = 5분 뒤. "예약 만들기" 누르면 5분 뒤 자동 발송됩니다. Just one thing — Target = "role", Kind = "once", Time = 5 minutes from now. Click "Create schedule" and it fires 5 min later.
  • 대상 종류 — 역할(예: res) 또는 플랫폼:채널(예: discord:1234567890) 중 선택.
  • Target kind — pick role (e.g. res) or platform:channel (e.g. discord:1234567890).
  • 대상 입력.
  • Enter the target.
  • 일정 종류 — "한 번(once)" 인지 "반복(cron)" 인지 선택.
  • Schedule kind — once or recurring (cron).
  • "예약 만들기" 클릭 → 통계 카드(대기 / 발송 / 실패 / 취소) 와 목록에 즉시 반영.
  • Click "Create schedule" → stats (pending / sent / failed / cancelled) and list update right away.
팁Tip — 반복 시각은 한국 시간(KST) 기준입니다. 서버가 UTC 라도 한국 시간대로 입력하면 됩니다.Recurring times are in KST. Even if your server runs UTC, type the time as KST.

같은 작업의 명령어: xgram schedule add · xgram schedule list · xgram schedule cancel <id>

Same as command: xgram schedule add · xgram schedule list · xgram schedule cancel <id>

탭 8 · 체인 / Chain

8. 체인 — 도미노 메시지

8. Chain — domino workflow

이 탭의 역할 — 여러 단계를 자동으로 차례대로 실행합니다. 도미노 — 첫 칸을 치면 끝까지 자동 진행. 각 단계의 성공·실패·건너뜀이 따로 기록됩니다.

What it does — runs multiple steps in order automatically. Like dominoes — tip the first one, the rest follow. Each step's success / failure / skip is tracked.

처음이라면: First time? 이 한 가지만 — 입력창의 예시 YAML 그대로 두고 "체인 생성", 그 뒤 "실행 (테스트 모드)" 누르세요. 어떤 단계가 어떻게 진행되는지 안전하게 볼 수 있습니다. Just one thing — keep the example YAML, click "Create chain", then "Run (dry-run)". You'll see each step in safe mode without real side-effects.
  • 입력창에 단계 정의(YAML — 사람이 읽기 쉬운 설정 형식기술 용어)를 작성합니다.
  • Write step definitions in YAML (a human-readable config formattech term).
  • "체인 생성" 버튼 → 목록에 추가됩니다.
  • Click "Create chain" → it joins the list.
  • "실행 (테스트 모드)" 으로 어떻게 흐르는지 확인 — 단계마다 실행됨 / 건너뜀 / 오류 표시.
  • "Run (dry-run)" to inspect — per-step executed / skipped / error.

같은 작업의 명령어: xgram chain create -f morning.yaml · xgram chain run morning · xgram chain list

11 · 문제 해결

11. 문제 해결

11. Troubleshooting

증상 — GUI 가 안 켜집니다 / 빈 창만 떠요 Symptom — GUI doesn't launch / blank window

원인: 화면 그리기 라이브러리(libwebkit2gtk-4.1) 가 없거나, 그래픽 가속이 안 되는 환경입니다. 화면이 없는 서버라면 정상 — 데스크톱에서 실행하세요. 일반 데스크톱이라면 libwebkit2gtk-4.1 을 설치하고 다시 켜보세요.

Cause: missing display library libwebkit2gtk-4.1 or no graphics acceleration. Expected on headless servers — run on a desktop. Otherwise install libwebkit2gtk-4.1 and retry.

증상 — 탭을 눌러도 반응이 없어요 Symptom — tab click does nothing

원인: 백그라운드에서 도는 OpenXgram 데몬이 꺼져 있거나, 포트가 다른 프로그램과 겹쳤습니다. 터미널에서 xgram status 로 확인하고, 필요하면 xgram daemon restart 로 다시 켜세요.

Cause: the background daemon is down or has a port conflict. Check with xgram status; restart with xgram daemon restart if needed.

증상 — 기억 검색 결과가 항상 비어 있어요 Symptom — Memory search always empty

원인: 검색용 작은 AI 모델(약 120MB) 이 아직 안 받아졌습니다. 첫 검색 때 자동으로 받아집니다. 회사·학교 같은 방화벽 환경이면 미리 받은 파일로 설치할 수 있어요: xgram model fetch --offline-tarball ./bge-small.tar.gz

Cause: the small search model (~120MB) hasn't been downloaded yet. Auto-downloads on first search. Behind a firewall, install offline: xgram model fetch --offline-tarball ./bge-small.tar.gz

증상 — 알림에서 토큰 검사가 실패합니다 Symptom — Notify token validation fails

원인: 토큰이 잘못됐거나 인터넷이 막혀 있습니다. Telegram 은 https://api.telegram.org, Discord 는 https://discord.com/api 로 나가는 통신이 필요합니다.

Cause: bad token or blocked outbound. Telegram needs https://api.telegram.org, Discord needs https://discord.com/api.

증상 — 채널 탭의 최근 메시지가 안 바뀝니다 Symptom — Channel "recent" stays stale

원인: 자동 갱신이 아니고 수동입니다. "새로고침" 버튼을 누르거나, 다른 탭 갔다가 다시 돌아오면 새 데이터가 들어옵니다.

Cause: refresh is manual. Click "Refresh" or revisit the tab to fetch.

증상 — 예약된 시각이 예상과 달라요 Symptom — Schedule fires at unexpected time

원인: 모든 시각은 한국 시간(KST) 기준입니다. 서버가 UTC 라면 9 시간 차이가 생기는데, 입력은 그냥 한국 시간 그대로 적으면 됩니다.

Cause: all times are KST. If your server is UTC, that's a 9-hour offset — just enter KST as-is.
