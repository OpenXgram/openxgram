인터페이스 선택 — CLI · TUI · GUI

Pick your interface — CLI · TUI · GUI

같은 데몬, 같은 ~/.openxgram/ 데이터. 어떤 인터페이스로 들어가도 결과는 같습니다. 상황에 맞춰 고르세요.

One daemon, one ~/.openxgram/. Whichever surface you pick, the result is the same. Choose by context.

CLI

xgram <cmd> — 자동화·CI·파이프

xgram <cmd> — automation, CI, pipes

  • 스크립트·AI 하네스(Claude Code · Cursor · Aider) 통합
  • Scripts, AI harness (Claude Code · Cursor · Aider)
  • SSH·서버·CI 환경에서 그대로 작동
  • Works on SSH / servers / CI
  • 다른 도구와 stdin/stdout 으로 연결
  • Composable via stdin/stdout

TUI

xgram tui — 터미널 안의 대시보드

xgram tui — dashboard inside your terminal

  • GPU·GUI 라이브러리 없는 환경 (서버·SSH)
  • No-GPU / no-GUI environments (servers, SSH)
  • 설치 manifest 빠른 확인
  • Quick install-state check
  • 키바인딩 Q/Esc 종료
  • Quit with Q/Esc

GUI

Tauri desktop app — 9-tab visual interface

  • 처음 사용자 · 봇 토큰 마법사 · 라이브 채널 stream
  • First-time users · bot wizard · live channel stream
  • Vault 30초 reveal 카운트다운, MFA 다이얼로그
  • Vault 30s reveal countdown, MFA dialog
  • 같은 데몬 / 같은 데이터 / 같은 결과
  • Same daemon / same data / same result
실용 가이드 — Practical guide — 처음 한 번은 GUI 로 봇 등록·신원 확인을 끝내고, 일상은 CLI 로 자동화. 서버에선 TUI 로 상태 점검. Bootstrap once with GUI (bot setup · identity), automate daily flows with CLI, peek at server state with TUI.

Tab 1 · 시작하기 / Onboarding

1. 시작하기 — init 진행 상태

1. Onboarding — init progress

이 탭은 무엇을 하나요 — 첫 실행 환영, init 진행 상태 확인, Memory 탭으로 자동 진행.

What this tab does — first-run welcome, init readiness check, auto-advance to Memory.

어떻게 쓰나요

How to use

  • 터미널에서 xgram init --alias <이름> 실행 (seed 와 ~/.openxgram/openxgram.db 생성).
  • Run xgram init --alias <name> in your terminal (creates seed + ~/.openxgram/openxgram.db).
  • BIP39 12 단어 시드를 안전한 곳에 적어둡니다 — 잃으면 vault 복구 불가.
  • Write the BIP39 12-word seed somewhere safe — losing it means no vault recovery.
  • GUI 의 "초기화 상태 다시 확인" 버튼을 눌러 초기화 완료 배지를 확인.
  • Click "Re-check initialization" until you see the Initialized badge.
  • (선택) Discord/Telegram 토큰을 알림 (Notify) 탭에서 연결.
  • (Optional) Connect Discord/Telegram in the Notify tab.
팁Tip — 이미 초기화된 경우 GUI 첫 화면이 자동으로 Memory 탭으로 전환됩니다.If already initialized, the GUI auto-switches to the Memory tab on launch.
주의Caution — seed 는 화면에 평문으로 노출됩니다. 화면 녹화·공유·스크린샷 주의.The seed is shown as plaintext. Beware of screen recording / sharing / screenshots.

같은 작업의 CLI: xgram init --alias myname · xgram status

Same in CLI: xgram init --alias myname · xgram status

Tab 2 · 기억 / Memory

2. 기억 — 5 층 메모리 검색

2. Memory — 5-layer search

이 탭은 무엇을 하나요 — L0 메시지·L2 기억·L4 정체성을 한 번에 검색.

What this tab does — search L0 messages, L2 memories, and L4 traits in one box.

  • 상단 입력창에 키워드 입력 (300ms 디바운스).
  • Type a keyword (300ms debounce).
  • L0 / L2 / L4 체크박스로 레이어 범위 조절.
  • Toggle L0 / L2 / L4 checkboxes to scope layers.
  • 점수(score) 내림차순으로 결과 확인 — 각 줄에 layer + score + body.
  • Scan score-sorted results — each row shows layer + score + body.
팁Tip — 여러 층을 비교하고 싶다면 L0+L2 만 켜고 검색해 시간 흐름을 확인하세요.To inspect a timeline, enable L0+L2 only.

같은 작업의 CLI: xgram session recall <키워드>

Same in CLI: xgram session recall <keyword>

Tab 3 · Vault

3. Vault — 자격증명 승인 + 30초 Reveal

3. Vault — credential approval + 30s reveal

이 탭은 무엇을 하나요 — 다른 에이전트가 요청한 자격증명을 승인/거부하고, 직접 vault 값을 reveal.

What this tab does — approve or deny credential requests from other agents, and reveal vault values directly.

  • 상단 Pending 카드에서 요청자(agent) · 요청 키 · 요청 시각 확인 후 승인/거부.
  • In the Pending card review requester · key · timestamp, then approve or deny.
  • 거부 시 사유 입력 (감사 로그에 기록).
  • When denying, enter a reason (logged to audit chain).
  • 하단 Vault Reveal 에 키 입력 → 30초간 마스킹 해제.
  • In Vault Reveal enter the key → unmasked for 30 seconds.
  • 클립보드 복사 시 30초 후 자동 클리어.
  • Copying to clipboard auto-clears after 30 seconds.
주의Caution — 승인 다이얼로그에서 fingerprint 가 알고 있는 에이전트의 것과 일치하는지 반드시 확인.Always verify the fingerprint in the approval dialog matches the agent you trust.

같은 작업의 CLI: xgram vault pending-list · xgram vault approve <id> · xgram vault reveal <key>

Same in CLI: xgram vault pending-list · xgram vault approve <id> · xgram vault reveal <key>

Tab 4 · 피어 / Peers

4. 피어 — 다른 머신·에이전트 등록

4. Peers — register other machines / agents

이 탭은 무엇을 하나요 — 다른 머신·에이전트의 신원을 등록하고 fingerprint 로 검증.

What this tab does — register other machines / agents and verify by fingerprint.

  • 상대 alias · address (http://… 또는 nostr://…) · public key (hex) 입력.
  • Enter peer alias · address (http://… or nostr://…) · public key (hex).
  • "피어 추가" 클릭 시 fingerprint 다이얼로그 표시 — 다른 채널(전화·종이)로 fingerprint 일치 확인 후 confirm.
  • Click "Add peer" → fingerprint dialog. Verify on a different channel (phone, paper), then confirm.
  • (선택) machine 입력 — ACL 화이트리스트로 사용.
  • (Optional) enter machine — used as ACL whitelist key.
주의Caution — fingerprint 미검증으로 추가하면 MITM 위험. 신원이 의심되면 거부.Adding without fingerprint check is MITM-prone. Deny anything you can't verify.

같은 작업의 CLI: xgram peer add --alias eno --address http://… --pubkey 0x… · xgram peer list

Same in CLI: xgram peer add --alias eno --address http://… --pubkey 0x… · xgram peer list

Tab 5 · 알림 / Notify

5. 알림 — Discord / Telegram 봇 마법사

5. Notify — Discord / Telegram bot wizard

이 탭은 무엇을 하나요 — 봇 토큰 검증 → chat_id 자동 감지 → 저장 → 테스트 메시지를 클릭 한 번으로.

What this tab does — validate bot token → auto-detect chat_id → save → send test, all in one click.

  • Telegram: BotFather 에서 받은 토큰 붙여넣기 → "토큰 검증" → bot username 표시 확인.
  • Telegram: paste BotFather token → "Validate" → confirm bot username.
  • 봇에게 직접 /start 메시지 보낸 뒤 "내 chat_id 자동 감지" 클릭.
  • Send /start to the bot, then click "Auto-detect chat_id".
  • "저장 + 테스트 전송" 클릭 → 봇에서 "OpenXgram GUI 연결 성공 ✓" 수신 확인.
  • Click "Save + send test" → check the bot for "OpenXgram GUI connected ✓".
  • Discord 도 동일 — 채널 ID 또는 Webhook URL 추가 입력 가능.
  • Discord follows the same flow — optional channel ID or webhook URL.
팁Tip — 토큰은 vault 에 저장됩니다. 평문 파일·환경변수에 두지 마세요.Tokens land in the vault. Don't keep them in plaintext files / env vars.

같은 작업의 CLI: xgram notify setup-telegram · xgram notify setup-discord

Same in CLI: xgram notify setup-telegram · xgram notify setup-discord

Tab 6 · 채널 / Channel

6. 채널 — 다중 에이전트 허브 라이브

6. Channel — live multi-agent hub

이 탭은 무엇을 하나요 — 어댑터 상태, 등록 피어 수, 예약 대기, 최근 메시지 20건 (KST 타임스탬프) 라이브.

What this tab does — adapter status, peer count, pending schedules, last 20 messages with KST timestamps — live.

  • "새로고침" 클릭으로 status + recent 동시 갱신.
  • Click "Refresh" to refetch status + recent messages.
  • 어댑터별 연결됨 / 미설정 배지 확인.
  • Per-adapter configured / not set badges.
  • "피어 수" · "예약 대기" 카드로 허브 활동량 한눈에.
  • Quick "Peers" / "Schedule pending" cards for hub-activity glance.

같은 작업의 CLI: xgram channel status · xgram channel recent --limit 20

Same in CLI: xgram channel status · xgram channel recent --limit 20

Tab 7 · 예약 / Schedule

7. 예약 — once / cron 메시지

7. Schedule — once / cron messages

이 탭은 무엇을 하나요 — 한 번(once) 또는 반복(cron) 으로 메시지 예약.

What this tab does — schedule a message once or recurring (cron, KST).

  • 대상 종류 선택 — 역할명(role) 또는 플랫폼:채널.
  • Pick target kind — role or platform:channel.
  • 대상 입력 (예: res 또는 discord:1234567890).
  • Enter target (e.g. res or discord:1234567890).
  • 스케줄 종류 선택 — once (ISO 시각) 또는 cron (KST 표현).
  • Pick kind — once (ISO datetime) or cron (KST expression).
  • "예약 만들기" 클릭. 통계 카드 (대기/발송/실패/취소) 와 목록에 즉시 반영.
  • Click "Create schedule". Stats (pending/sent/failed/cancelled) and the list update immediately.
팁Tip — cron 표현은 KST 기준입니다. 서버 TZ 가 UTC 여도 OpenXgram 데몬이 KST 로 해석.Cron expressions are interpreted in KST. Even if your server runs UTC, the daemon evaluates KST.

같은 작업의 CLI: xgram schedule add · xgram schedule list · xgram schedule cancel <id>

Same in CLI: xgram schedule add · xgram schedule list · xgram schedule cancel <id>

Tab 8 · 체인 / Chain

8. 체인 — YAML 워크플로우

8. Chain — YAML workflows

이 탭은 무엇을 하나요 — 여러 단계를 YAML 로 정의해 한 번에 실행 (각 단계 응답·실패·skip 추적).

What this tab does — define multi-step workflows in YAML and run them with per-step result tracking.

  • 새 체인 입력창에 YAML 작성 (예: name: morning + steps:).
  • Write YAML in the new-chain box (e.g. name: morning + steps:).
  • "체인 생성" 클릭 → 목록에 추가.
  • Click "Create chain" → it joins the list.
  • 체인의 "실행 (dry-run)" 으로 결과 확인 — 각 step 마다 executed/skipped/error 표시.
  • Run "Run (dry-run)" to inspect — per-step executed/skipped/error.

같은 작업의 CLI: xgram chain create -f morning.yaml · xgram chain run morning · xgram chain list

Same in CLI: xgram chain create -f morning.yaml · xgram chain run morning · xgram chain list

Tab 9 · 설정 / Settings

9. 설정 — USDC 일일 한도

9. Settings — USDC daily limit

이 탭은 무엇을 하나요 — USDC 결제 일일 한도 확인·변경. 변경 시 MFA 재인증.

What this tab does — view and change the USDC daily payment cap. Changes require MFA re-auth.

  • 현재 한도 확인 (USDC 단위).
  • Read the current cap (USDC).
  • 새 한도 입력 (소수점 2자리 가능) → 확인.
  • Enter new cap (2 decimal places) → Confirm.
  • MFA 재인증 다이얼로그에서 진행 (PRD-MFA-02 통합 후 WebAuthn).
  • Approve in the MFA re-auth dialog (WebAuthn once PRD-MFA-02 lands).
주의Caution — 한도 0 으로 설정하면 모든 결제가 차단됩니다 (의도적인 kill-switch).Setting cap to 0 blocks all payments (intentional kill-switch).

같은 작업의 CLI: xgram payment limit get · xgram payment limit set 50.00

Same in CLI: xgram payment limit get · xgram payment limit set 50.00

11 · Troubleshooting

11. 트러블슈팅

11. Troubleshooting

증상 — GUI 실행 안 됨 / 빈 창 Symptom — GUI doesn't launch / blank window

원인: WebKit 의존 라이브러리(libwebkit2gtk-4.1) 부재 또는 GPU/DRI3 미지원. 헤드리스 서버에선 정상 — 데스크톱 환경에서 실행하세요. 일반 데스크톱이면 libwebkit2gtk-4.1 설치 후 재시도.

Cause: missing libwebkit2gtk-4.1 or no GPU/DRI3. Expected on headless servers — run on a desktop environment. Otherwise install libwebkit2gtk-4.1 and retry.

증상 — 탭 클릭 응답 없음 Symptom — tab click does nothing

원인: 데몬이 떠 있지 않거나 포트 충돌. xgram status 로 데몬 상태 확인. 필요하면 xgram daemon restart.

Cause: daemon not running or port conflict. Check xgram status; xgram daemon restart if needed.

증상 — Memory 검색 결과가 항상 비어 있음 Symptom — Memory search always empty

원인: BGE-small 모델이 다운로드되지 않음. 첫 회상 시 ~120MB 자동 다운로드. 방화벽 환경이면 xgram model fetch --offline-tarball ./bge-small.tar.gz 로 오프라인 설치.

Cause: BGE-small model missing. ~120MB auto-downloads on first recall. Behind a firewall use xgram model fetch --offline-tarball ./bge-small.tar.gz.

증상 — Notify 토큰 검증 실패 Symptom — Notify token validation fails

원인: 잘못된 토큰 또는 네트워크 차단. Telegram 은 https://api.telegram.org, Discord 는 https://discord.com/api outbound 가 필요합니다.

Cause: bad token or blocked outbound. Telegram needs https://api.telegram.org, Discord needs https://discord.com/api.

증상 — Channel 탭의 최근 메시지가 안 갱신 Symptom — Channel tab "recent" stale

원인: 자동 polling 대신 수동 갱신. 화면의 "새로고침" 버튼을 누르거나 다른 탭에서 돌아오면 다시 fetch 합니다.

Cause: refresh is manual. Click "Refresh" or revisit the tab to re-fetch.

증상 — Schedule 의 cron 시각이 예상과 다름 Symptom — Schedule cron fires at unexpected time

원인: cron 표현은 KST 기준입니다. 서버 TZ 가 UTC 면 9 시간 차이 — KST 그대로 입력하세요.

Cause: cron is interpreted in KST. If your server is UTC, that's a 9-hour offset — enter KST as-is.
