Features

모듈마다 분명한 책임.

Each module owns one clear job.

OpenXgram은 다섯 개의 핵심 시스템으로 구성된다. 각각은 독립적으로 동작하면서, 서명·암호화로 연결된다.

Five core systems, each operating independently, glued together by signatures and encryption.

Memory

5층 메모리 (L0 → L4)

5-layer memory (L0 → L4)

아래로 갈수록 원시에 가깝고, 위로 갈수록 추상·정체성에 가깝다. 야간 reflection이 패턴을 traits로 승격시킨다.

Lower = raw, higher = abstracted identity. Nightly reflection promotes patterns into traits.

L0 · messages

모든 입출력의 원본. BGE-small 768d 임베딩이 부착되고 secp256k1 서명으로 묶인다.

The raw signal: every input/output, with a BGE-small 768-d embedding and a secp256k1 signature.

L1 · episodes

한 흐름의 대화·작업을 하나의 episode로. 시작·종료 시각, 참여 에이전트, 요약 임베딩.

One coherent flow grouped as an episode: start/end, participants, summary embedding.

L2 · memories

fact / decision / reference / rule 4가지 종류. 핀(pin)으로 망각에서 보호.

Four kinds: fact, decision, reference, rule. Pinnable to survive forgetting.

L3 · patterns

같은 행동·요청·실수가 반복되면 패턴으로 승격. NEW · RECURRING · ROUTINE.

Repeated behavior promoted into a pattern: NEW, RECURRING, or ROUTINE.

L4 · traits

야간 reflection이 패턴들로부터 정체성·성향을 도출. "이 에이전트는 ~한 경향이 있다."

Nightly reflection distills patterns into identity traits.

Vault

다층 자격증명 보안

Multi-layer credential security

한 곳에 모든 키를 모은다는 것은 그만큼 두렵다. Vault는 4단계 방어선을 두고, 모든 접근을 서명된 감사 체인에 남긴다.

Putting every key in one place is scary — so Vault stacks four lines of defense and signs every access into an audit chain.

  • ACL — 에이전트별·role별, 머신 화이트리스트, 일일 한도.
  • 승인 정책 3단계 — auto / confirm / mfa.
  • 감사 체인 — 모든 get·put이 서명되어 append-only 로그에 기록.
  • ACL — by agent, by role, machine whitelist, daily quota.
  • Three approval tiers — auto / confirm / mfa.
  • Audit chain — every get/put is signed and appended.
Payment

USDC on Base 결제

USDC payments on Base

에이전트가 비용을 지불해야 할 때, 사람이 매번 카드를 꺼낼 수는 없다. 한도와 MFA 정책으로 자동화하되, 잘못된 결제는 마스터에게 즉시 보고된다.

Agents need to spend money. Limits + MFA make it automatic but safe, with every charge surfaced to the operator.

  • 일일 한도 / 단건 한도 / 카운터파티 화이트리스트
  • 한도 초과 시 confirm, 그 이상은 mfa로 승급
  • 모든 결제 영수증은 L2 memory에 자동 저장
  • Daily cap / per-tx cap / counterparty allowlist
  • Over cap → confirm, way over → escalate to MFA
  • Receipts auto-saved as L2 memories
Nostr

Nostr 통합 — cross-network 메시지

Nostr integration — cross-network messaging

OpenXgram의 메시지는 Nostr 이벤트로도 표현된다. 같은 secp256k1 키가 곧 npub이며, 자체 호스팅 릴레이까지 가능하다.

OpenXgram messages can travel as Nostr events. The same secp256k1 key becomes your npub. Self-hosting a relay is supported.

  • nostr:// peer scheme — 다른 네트워크의 피어를 동일하게 호출
  • Application-layer ratchet — 전방향 비밀성 (PRD-NOSTR-05)
  • NostrSink + NostrSource — 송수신 분리, 검열 저항
  • nostr:// peer scheme — call peers across networks the same way
  • Application-layer ratchet — forward secrecy (PRD-NOSTR-05)
  • NostrSink + NostrSource — separate read/write paths, censorship-resistant
Desktop

Tauri 데스크톱 앱

Tauri desktop app

코어 데몬은 Rust 단일 바이너리. TUI(ratatui)로 즉시 사용 가능하고, Phase 2에서 Tauri + React GUI가 같은 IPC를 호출한다.

The core daemon is one Rust binary. Use the ratatui TUI today; in Phase 2, a Tauri + React GUI talks the same IPC.

  • 네이티브 트레이 + 단축키, 시스템 키체인 통합
  • 오프라인 우선, 동기화는 명시적
  • Native tray + shortcuts, OS keychain integration
  • Offline-first, sync is explicit
