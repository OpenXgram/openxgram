왜 OpenXgram?
Why OpenXgram?

대화는 끊기고,
데이터는 새어나갑니다.

Conversations break.
Data leaks out.

Claude · ChatGPT · Gemini 사이를 오갈 때마다 컨텍스트가 사라지고, 기존 메모리 SaaS(Mem0·Zep 등)를 쓰면 사용자의 데이터가 그들의 서버에 임베딩으로 남습니다. OpenXgram 은 다릅니다 — 어떤 AI 든 같은 기억을 이어가지만, 모든 데이터가 사용자(나) 의 기기 안에서만 머무릅니다. 블록체인 기반 암호화로 외부 서버로 전송되지 않습니다.

Every time you move between Claude, ChatGPT, Gemini, the context vanishes — and existing memory SaaS (Mem0, Zep, etc.) embed your data on their own third-party servers. OpenXgram is different — the same memory carries across any AI, but all of it stays on your machine. Blockchain-grade encryption means nothing is ever sent to an external server.

pain · memory loss

"또 처음부터 설명해야 한다"

"I'm explaining everything again"

Claude Code 에서 진행하던 작업을 Cursor 로 옮기면 컨텍스트가 사라집니다. 다음 세션에 같은 자료를 또 붙여 넣게 됩니다.

Move from Claude Code to Cursor and the context evaporates. You re-paste the same files into the next session.

OpenXgram: L0~L4 다층 기억이 머신과 하네스 밖에 살아 있어, 어떤 도구에서 켜도 어제 했던 결정·노트·파일을 그대로 봅니다.

OpenXgram: the multi-layer memory lives outside any single harness, so any tool you open still sees yesterday's decisions, notes, files.

pain · scattered keys

"키가 .env 파일 곳곳에 흩어졌다"

"Keys are scattered across .env files"

레포마다 .env, OS 키체인, 1Password — 에이전트가 어떤 걸 읽고 쓰는지 추적이 안 됩니다. 토큰이 평문으로 컨텍스트에 떠다닙니다.

Every repo has its own .env, OS keychain, 1Password — there's no audit trail of what the agent actually used. Plain-text tokens drift into context.

OpenXgram vault: 4단계 암호화(디스크/필드/전송/메모리 zeroize) + ACL + 일일 한도 + 블록체인 암호화 기록. 키 자체는 에이전트 컨텍스트에 노출되지 않습니다.

OpenXgram vault: four-layer encryption + ACL + daily limits + blockchain-encrypted log. The raw key never enters the agent's context.

pain · 다른 서비스의 문제

"클라우드 메모리 서비스를 쓰면 내 데이터가 그들 서버로 간다"

"Cloud memory SaaS reads everything you store"

다른 서비스(Mem0 · Zep 등)를 쓸 경우 — 사용자가 저장한 메모리가 그들의 클라우드 서버로 전송됩니다. 회사 비밀·개인 기록·클라이언트 자료가 모두 제3자 서버에 임베딩 형태로 남고, 한 번 올라간 데이터는 통제할 수 없습니다.

If you use other services (Mem0, Zep, etc.) — your stored memory is sent to their cloud servers. Company secrets, personal notes, client data all end up embedded on a third-party server, and once uploaded, you lose control.

OpenXgram 은 정반대입니다. 모든 메모리·임베딩이 사용자(나) 의 기기 안 단일 SQLite 파일에만 저장됩니다. BGE-small 임베딩 모델도 로컬에서 실행됩니다. 외부 서버로 전송되는 데이터는 0 입니다. 다른 기기와 동기화할 때도 사용자가 직접 운영하는 Nostr relay 위에서 블록체인 암호화로 주고받습니다.

OpenXgram is the exact opposite. All memory and embeddings live in a single SQLite file on your own machine. The BGE-small embedding model runs locally. Zero data is transmitted to any external server. Even cross-device sync runs over your self-hosted Nostr relay with blockchain-grade encryption.

pain · runaway agent

"에이전트가 어디까지 할지 통제가 안 된다"

"There's no real boundary for the agent"

자율 에이전트에게 카드·API 키를 주는 순간 위험이 시작됩니다. 얼마까지 쓸지, 무엇은 사람 승인이 필요한지 — 명세할 곳이 없습니다.

The moment you hand a card or API key to an autonomous agent, risk starts. How much can it spend? What needs a human? There's nowhere to declare it.

OpenXgram policy: auto / confirm / mfa 3단계 승인 + 일일 한도 + 화이트리스트. 넘으려 하면 즉시 막히고 감사 로그에 남습니다.

OpenXgram policy: three-step approval (auto / confirm / mfa) + daily limit + whitelist. Crossings are blocked at the boundary and recorded in the blockchain-encrypted log.

pain · cross-machine

"같은 에이전트팀을 다른 머신에서 못 돌린다"

"My agent team can't move between machines"

노트북에 셋업한 멀티에이전트 시스템을 새 머신에서 돌리려면 키 재발급, 메모리 export, 설정 복제가 필요합니다.

Spinning up the same multi-agent system on a new machine means reissuing keys, exporting memory, copying configs.

OpenXgram: BIP44 HD wallet 한 번이면 같은 신원이 어디서나. Nostr 채널로 메모리 동기화. 머신 화이트리스트로 안전하게 합류.

OpenXgram: one BIP44 HD wallet, the same identity anywhere. Nostr channels sync the memory; a machine whitelist keeps the join safe.

pain · payment

"에이전트 결제가 거칠고 비싸다"

"Agent payments are clunky and expensive"

신용카드 결제는 한도 변경·환불·신원확인이 까다롭고, 수수료가 비싼 블록체인 네트워크에서는 micropayment 가 거의 불가능합니다.

Credit cards struggle with limits, refunds, KYC; networks with high fees make micropayments unrealistic.

OpenXgram: Base 네트워크 위 USDC — 1원 이하의 수수료로 micropayment 가 현실적입니다. 사용자(나) 가 정한 한도와 승인 절차 안에서 에이전트가 자율적으로 결제할 수 있습니다.

OpenXgram: USDC on Base — fees under 1 cent make micropayments practical. Agents pay autonomously within the limits and approval rules you set.
