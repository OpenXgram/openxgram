왜 OpenXgram?
Why OpenXgram?

대화는 끊기고,
데이터는 새어나갑니다.

Conversations break.
Data leaks out.

Claude · ChatGPT · Gemini 사이를 오갈 때마다 컨텍스트가 사라지고, 메모리 SaaS 는 당신의 데이터를 자기 서버에 임베딩합니다. OpenXgram 은 어떤 AI 든 같은 기억을 이어가게 하면서, 블록체인 기반 암호화로 데이터를 당신의 머신 밖으로 절대 내보내지 않습니다.

Every time you move between Claude, ChatGPT, Gemini, the context vanishes — and memory SaaS embeds your data on a third-party server. OpenXgram keeps the same memory across any AI, while blockchain-grade encryption ensures data never leaves your machines.

pain · memory loss

"또 처음부터 설명해야 한다"

"I'm explaining everything again"

Claude Code 에서 진행하던 작업을 Cursor 로 옮기면 컨텍스트가 사라집니다. 다음 세션에 같은 자료를 또 붙여 넣게 됩니다.

Move from Claude Code to Cursor and the context evaporates. You re-paste the same files into the next session.

OpenXgram: L0~L4 다층 기억이 머신과 하네스 밖에 살아 있어, 어떤 도구에서 켜도 어제 했던 결정·노트·파일을 그대로 봅니다.

OpenXgram: the multi-layer memory lives outside any single harness, so any tool you open still sees yesterday's decisions, notes, files.

pain · scattered keys

"키가 .env 파일 곳곳에 흩어졌다"

"Keys are scattered across .env files"

레포마다 .env, OS 키체인, 1Password — 에이전트가 어떤 걸 읽고 쓰는지 추적이 안 됩니다. 토큰이 평문으로 컨텍스트에 떠다닙니다.

Every repo has its own .env, OS keychain, 1Password — there's no audit trail of what the agent actually used. Plain-text tokens drift into context.

OpenXgram vault: 4단계 암호화(디스크/필드/전송/메모리 zeroize) + ACL + 일일 한도 + 감사 chain. 키 자체는 에이전트 컨텍스트에 노출되지 않습니다.

OpenXgram vault: four-layer encryption + ACL + daily limits + audit chain. The raw key never enters the agent's context.

pain · cloud lock-in

"메모리 SaaS 가 내 데이터를 본다"

"My memory SaaS reads everything"

Mem0·Zep 등은 메모리를 클라우드에 저장합니다. 회사 비밀, 개인 일기, 클라이언트 자료가 제3자 서버에 임베딩되어 들어갑니다.

Mem0, Zep et al. store memory in the cloud. Company secrets, journals, client data end up embedded on a third-party server.

OpenXgram: 단일 SQLite 파일 + sqlite-vec, 임베딩(BGE-small)도 로컬. Nostr 동기화도 자체 호스팅 relay 가능. 데이터가 사용자 기기를 떠나지 않습니다.

OpenXgram: a single SQLite file + sqlite-vec, embeddings (BGE-small) run locally. Even Nostr sync can use a self-hosted relay. Data never leaves your machine.

pain · runaway agent

"에이전트가 어디까지 할지 통제가 안 된다"

"There's no real boundary for the agent"

자율 에이전트에게 카드·API 키를 주는 순간 위험이 시작됩니다. 얼마까지 쓸지, 무엇은 사람 승인이 필요한지 — 명세할 곳이 없습니다.

The moment you hand a card or API key to an autonomous agent, risk starts. How much can it spend? What needs a human? There's nowhere to declare it.

OpenXgram policy: auto / confirm / mfa 3단계 승인 + 일일 한도 + 화이트리스트. 넘으려 하면 즉시 막히고 감사 로그에 남습니다.

OpenXgram policy: three-step approval (auto / confirm / mfa) + daily limit + whitelist. Crossings are blocked at the boundary and recorded in the audit chain.

pain · cross-machine

"같은 에이전트팀을 다른 머신에서 못 돌린다"

"My agent team can't move between machines"

노트북에 셋업한 멀티에이전트 시스템을 새 머신에서 돌리려면 키 재발급, 메모리 export, 설정 복제가 필요합니다.

Spinning up the same multi-agent system on a new machine means reissuing keys, exporting memory, copying configs.

OpenXgram: BIP44 HD wallet 한 번이면 같은 신원이 어디서나. Nostr 채널로 메모리 동기화. 머신 화이트리스트로 안전하게 합류.

OpenXgram: one BIP44 HD wallet, the same identity anywhere. Nostr channels sync the memory; a machine whitelist keeps the join safe.

pain · payment

"에이전트 결제가 거칠고 비싸다"

"Agent payments are clunky and expensive"

카드 결제는 한도·취소·KYC 가 어렵고, 가스비 높은 체인은 micropayment 가 비현실적입니다.

Cards struggle with limits, refunds, KYC; high-gas chains make micropayments unrealistic.

OpenXgram: USDC on Base — 가스 거의 0, EVM 표준. vault 한도와 승인 정책으로 자율 결제가 가능하면서도 안전합니다.

OpenXgram: USDC on Base — near-zero gas, EVM standard. Vault limits + approval policy make autonomous payment safe.
